# V2.1 Multi-Agent Architecture

> **状态:** 已完成 | **日期:** 2026-06-15 | **基于:** v2 Web 应用

## 1. 目标

将 v2 的单线程 Research 节点替换为 4 个并行 Agent（Paper / GitHub / Blog / Docs），通过 LangGraph Send API 扇出执行，Merge 节点收集并交叉验证结果，可选 Human-in-the-Loop 审核。

## 2. Graph 拓扑（V2.1）

```text
START → plan (增强：LLM 输出 source_types)

plan --[Send API 扇出]--> research_agent × N (并发)
    ├── profile: "paper"  ─── site:arxiv.org + 学术关键词
    ├── profile: "github" ─── site:github.com + 代码关键词
    ├── profile: "blog"   ─── 技术博客偏向
    └── profile: "docs"   ─── site:docs.* + 官方文档关键词

research_agent × N --[auto-barrier]--> merge (三阶段流水线)

merge --> human_review (可选，LangGraph interrupt)
human_review --> summary (复用 v2)
summary --> critique (复用 v2)
critique --[pass/fail]--> final | loop back to research_agent
final --> END
```

### 与 v2 拓扑对比

| 节点 | v2 | v2.1 | 变化 |
|------|-----|------|------|
| `plan` | 生成 sub_questions | 额外输出 source_types | 增强 |
| `research` | 单线程搜索+抓取+提取 | **移除** | 替换 |
| `research_agent` | — | **新增** — 统一节点 + Profile 注入 | 新增 |
| `merge` | — | **新增** — 三阶段流水线 | 新增 |
| `human_review` | — | **新增** — LangGraph interrupt | 新增 |
| `summary` | 综合 evidence → draft | 不变 | 复用 |
| `critique` | 三维评分 | 不变（接受 merge_summary conflicts） | 复用 |
| `final` | 生成最终报告 | 不变 | 复用 |

## 3. 核心新增模块

### 3.1 AgentProfile（research_agent.py）

```
AgentProfile (dataclass)
├── name: str           — "学术论文 Agent"
├── search_modifiers    — ["site:arxiv.org", "site:scholar.google.com"]
├── system_prompt       — Agent 专属 prompt
└── evidence_instruction — 证据提取偏好指令

AGENT_PROFILES = {paper, github, blog, docs}
```

4 个 Agent 共享同一个 `make_research_agent(llm)` 工厂函数，差异仅在于 `AgentProfile` 配置。Send API 通过 `state["agent_profile"]` 注入 profile key。

### 3.2 扇出路由（graph.py）

```python
def fanout_to_agents(state) -> list[Send]:
    for sq in state["research_plan"]["sub_questions"]:
        for st in sq.get("source_types", ["blog"]):
            yield Send("research_agent", {
                "agent_profile": st,
                "sub_question": sq,
            })
```

- 每个 `(sub_question, source_type)` 对生成一个 `Send`
- LangGraph 运行时并行执行所有 Send，全部完成后自动 barrier 到 merge
- Critique follow-up 阶段：new_search_queries 发往全部 4 个 Agent

### 3.3 Merge 三阶段流水线（merge.py）

```
Stage 1: Collect & Normalize
  └── 汇总 sources + evidences，标注 source_agent

Stage 2: Dedup & Cross-Validate
  ├── URL 精确去重 sources
  ├── LLM 语义去重 evidences（复用 dedup.py）
  ├── 交叉验证 (cross_validate.py) — 多 Agent 确认 → +confidence
  ├── 冲突检测 (detect_conflicts) — 矛盾 claim → flag
  └── Source 权威度排序（复用 ranking.py）

Stage 3: Quality Report
  └── MergeSummary { sources, evidences, cross_validated, conflicts, bias_warnings }
```

### 3.4 交叉验证（cross_validate.py）

- **cross_validate_evidences()**: O(n²) LLM 聚类 → 多 Agent 一致 +0.05～0.2 confidence；独有发现标记 source_bias
- **detect_conflicts()**: 聚类内 LLM 矛盾判定 → `[{topic, positions, severity}]` 输出

### 3.5 Human-in-the-Loop（human_review.py）

```python
human_decision = interrupt({merge_summary, sources, evidences, conflicts})
# Graph 暂停，等待 POST /api/tasks/{id}/review 提交决策
```

- **approve**: 继续 → summary → critique → final
- **amend**: 注入 new_search_queries → 回到 research_agent
- **redo**: 设置 error → 终止流程
- 默认禁用 (`human_review_enabled: bool = False`)，通过 task 参数按需启用

## 4. State 扩展

AgentState 新增字段：

| 字段 | 类型 | 用途 |
|------|------|------|
| `agent_outputs` | `list[dict]` | 每个 Agent 的中间产出 |
| `merge_summary` | `dict | None` | MergeSummary 质量报告 |
| `human_review` | `dict | None` | 人工审核决策 |
| `agent_profile` | `str | None` | Send API 注入的 profile key |
| `sub_question` | `dict | None` | Send API 注入的单个子问题 |

Pydantic 模型新增：

| 模型 | 新增字段 | 用途 |
|------|---------|------|
| `SubQuestion` | `source_types: list[str] = ["blog"]` | LLM 分类输出 |
| `Source` | `source_agent: str = "unknown"` | 来源 Agent 标记 |
| `Evidence` | `source_agent: str = "unknown"` | 来源 Agent 标记 |
| `MergeSummary` | (新模型) | 合并质量报告 |

## 5. API 扩展（Phase 3）

```
POST /api/tasks/{task_id}/review
  Request: {action: "approve"|"amend"|"redo", notes: str, new_queries: [...]}
  Response: {status: "ok", task_id, action}
```

SSE 事件新增：`review_required` — 携带 merge_summary + sources + evidences + conflicts，触发前端 ReviewPanel。

## 6. 设计决策

| # | 决策点 | 选择 | 理由 |
|---|--------|------|------|
| D1 | 并发实现 | LangGraph Send API | Checkpoint/Streaming 原生支持 |
| D2 | Agent 粒度 | 统一节点 + Profile 注入 | 行为相似，避免过度抽象 |
| D3 | 问题路由 | Plan LLM 分类 | 零额外成本，支持交叉覆盖 |
| D4 | 审核位置 | Merge 之后 | 信息最充分，干预时机最佳 |
| D5 | 审核实现 | LangGraph interrupt() | 原生暂停/恢复，无需外部状态管理 |
