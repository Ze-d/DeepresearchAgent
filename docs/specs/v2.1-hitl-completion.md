# V2.1 HITL 完整实现计划

> **状态:** 规划中 | **日期:** 2026-06-15 | **依赖:** V2.1 多 Agent 架构已完成

## 1. 背景

V2.1 Phase 3 实现了 HITL 的基础设施（`human_review` 节点 + `/review` endpoint + `ReviewPanel.vue`），但存在一个关键缺口：**LangGraph `interrupt()` 的暂停/恢复机制未与上层调用方对接**。

### 1.1 当前已实现

| 组件 | 文件 | 状态 |
|------|------|------|
| `human_review_node` | [deepresearch/nodes/human_review.py](../deepresearch/nodes/human_review.py) | ✅ 完整 — approve/amend/redo 三种决策，含 `interrupt()` 调用 |
| `POST /api/tasks/{id}/review` | [server/routes.py](../server/routes.py#L184-L201) | ✅ 端点就绪 — 接收审核决策并写入 task_manager |
| `ReviewPanel.vue` | [web/src/components/](../web/src/components/) | ✅ 前端就绪 — 展示 merge_summary + sources + conflicts |
| `human_review_enabled` 配置 | [deepresearch/config.py](../deepresearch/config.py#L58) | ✅ 开关就绪 — 默认 `False` |
| Graph 拓扑 | [deepresearch/graph.py](../deepresearch/graph.py#L120) | ✅ merge → human_review → summary 边就绪 |

### 1.2 缺失的环节

```
┌─ 当前工作流 ─────────────────────────────────────────────────┐
│                                                               │
│  Server:  invoke() ──→ human_review ──→ interrupt()          │
│                       抛出 GraphInterrupt                      │
│                            ↓                                  │
│                       被 except Exception 捕获                  │
│                            ↓                                  │
│                       任务标记为 FAILED 💥                      │
│                                                               │
│  CLI:     stream("updates") ──→ human_review ──→ interrupt()  │
│                       同样抛出 GraphInterrupt，无人处理          │
│                                                               │
└───────────────────────────────────────────────────────────────┘

┌─ 目标工作流 ─────────────────────────────────────────────────┐
│                                                               │
│  Server:  astream() ──→ human_review ──→ interrupt()          │
│                            ↓                                  │
│                   捕获 GraphInterrupt                          │
│                            ↓                                  │
│                   任务状态 → REVIEW_REQUIRED                    │
│                   推送 SSE: review_required                    │
│                            ↓                                  │
│                   用户点击 approve/amend/redo                   │
│                            ↓                                  │
│                   POST /api/tasks/{id}/review                 │
│                            ↓                                  │
│                   Command(resume=...) 恢复执行                  │
│                                                               │
│  CLI:     stream() + 捕获 GraphInterrupt                       │
│                        → Rich prompt 交互输入                  │
│                        → Command(resume=...) 继续              │
│                                                               │
└───────────────────────────────────────────────────────────────┘
```

## 2. 根因分析

### 2.1 问题 1：默认关闭

```python
# deepresearch/config.py:58
human_review_enabled: bool = False
```

`human_review_node` 在 `human_review_enabled=False` 时直接返回 `auto-approve`，**从不调用 `interrupt()`**。这是合理的默认值——在 HITL 完整实现前避免崩溃。

### 2.2 问题 2：Server 模式不支持 interrupt/resume

`server/routes.py:50` 在后台线程中调用 `run_workflow()`：

```python
result = run_workflow(req.query, max_iterations=req.max_iterations)
```

而 `run_workflow()` → `cli.py:63` 使用 `app_graph.invoke()`：

```python
result = app_graph.invoke(initial_state, config)
```

**`invoke()` 不支持 `interrupt()`**。当 `interrupt()` 被调用时，LangGraph 抛出 `GraphInterrupt` 异常，被 `routes.py:76` 的 `except Exception` 捕获，任务标记为 FAILED。

正确的做法是使用 `graph.astream()` 或 `graph.stream()` 并捕获 `GraphInterrupt`，保存 checkpoint，等待人工输入后通过 `Command(resume=...)` 恢复。

### 2.3 问题 3：CLI stream 模式同样不支持

`streaming/renderer.py:108` 使用 `graph.stream(stream_mode="updates")`：

```python
for chunk in graph.stream(initial_state, config, stream_mode="updates"):
```

`stream_mode="updates"` 在每个 superstep 完成后 yield 结果。如果中途 `interrupt()` 被调用，`stream()` 会抛出 `GraphInterrupt`——当前代码未捕获此异常，会导致 CLI 崩溃。

### 2.4 问题 4：review endpoint 不恢复执行

`server/routes.py:184-201` 的 `/review` endpoint 只将决策写入 `task_manager`，**未调用 `Command(resume=...)` 恢复 graph 执行**。

```python
@router.post("/tasks/{task_id}/review")
def submit_review(task_id: str, req: ReviewRequest):
    task_manager.update(task_id,
        status=TaskStatus.RUNNING.value,  # 改回 RUNNING，但 graph 实际已死
        review_decision={...})
    return {"status": "ok", "task_id": task_id, "action": req.action}
```

### 3.2 Task H1.1：重构 Server 执行引擎

**目标：** 将后台任务从 `invoke()` 切换为 `astream()`，捕获 `GraphInterrupt`。

**文件：** `server/routes.py`、`server/tasks.py`

**改动：**

```python
# server/routes.py — 新的 run_background 协程
import asyncio
from langgraph.errors import GraphInterrupt

async def run_background(task_id: str, query: str, max_iterations: int):
    """在后台异步执行 workflow，支持 HITL interrupt/resume。"""
    from deepresearch.cli import _make_initial_state
    from deepresearch.graph import build_graph
    from deepresearch.checkpoint.manager import CheckpointManager
    from deepresearch.output import init_session_dir

    graph = build_graph()
    session_dir = init_session_dir()
    cm = CheckpointManager(session_dir)
    app_graph = graph.compile(checkpointer=cm.saver)
    config = {"configurable": {"thread_id": session_dir.name}}

    initial_state = _make_initial_state(query, max_iterations)
    # 注入 human_review_enabled
    initial_state["human_review_enabled"] = True

    try:
        task_manager.update(task_id, status=TaskStatus.RUNNING.value)
        sse_manager.put_event(task_id, "task_started", {"task_id": task_id})

        async for chunk in app_graph.astream(initial_state, config):
            for node_name, node_result in chunk.items():
                sse_manager.put_event(task_id, "node_done", {
                    "node": node_name,
                    "result": _serialize_node_result(node_name, node_result),
                })

        # 正常完成
        final_state = app_graph.get_state(config)
        # ... 保存结果，标记 COMPLETED ...

    except GraphInterrupt as gi:
        # HITL: Graph 暂停，等待人工审核
        interrupt_data = gi.args[0] if gi.args else {}
        task_manager.update(task_id,
            status=TaskStatus.REVIEW_REQUIRED.value,
            interrupt_data=interrupt_data,
            checkpoint_config=config,  # 保存 config 以便恢复
        )
        sse_manager.put_event(task_id, "review_required", interrupt_data)
        logger.info("Task %s waiting for human review", task_id)

    except Exception as e:
        logger.exception("Task %s failed", task_id)
        task_manager.update(task_id, status=TaskStatus.FAILED.value, error=str(e))
        sse_manager.put_event(task_id, "error", {"error": str(e)})

# 在 create_task 中启动
@router.post("/tasks", status_code=202)
async def create_task(req: CreateTaskRequest):
    task = task_manager.create(req.query, max_iterations=req.max_iterations)
    asyncio.create_task(run_background(task["task_id"], req.query, req.max_iterations))
    return {"task_id": task["task_id"], "status": task["status"]}
```

**关键变化：**
1. `threading.Thread` → `asyncio.create_task`（LangGraph `astream` 需要 async）
2. `invoke()` → `astream()`（支持 interrupt/resume）
3. 捕获 `GraphInterrupt` → 保存 checkpoint config → 任务状态 `REVIEW_REQUIRED`
4. SSE 事件 `review_required` 推送给前端

### 3.3 Task H2：Review Endpoint 恢复执行

**目标：** `/review` endpoint 接收决策后，通过 `Command(resume=...)` 恢复 graph。

**文件：** `server/routes.py`

**改动：**

```python
from langgraph.types import Command

@router.post("/tasks/{task_id}/review")
async def submit_review(task_id: str, req: ReviewRequest):
    task = task_manager.get(task_id)
    if task is None:
        raise HTTPException(status_code=404, detail="Task not found")
    if task["status"] != TaskStatus.REVIEW_REQUIRED.value:
        raise HTTPException(status_code=400, detail="Task is not awaiting review")

    # 保存审核决策
    decision = {"action": req.action, "notes": req.notes, "new_queries": req.new_queries}
    task_manager.update(task_id,
        status=TaskStatus.RUNNING.value,
        review_decision=decision,
    )

    # 恢复 graph 执行
    config = task["checkpoint_config"]
    asyncio.create_task(resume_graph(task_id, config, decision))

    return {"status": "ok", "task_id": task_id, "action": req.action}


async def resume_graph(task_id: str, config: dict, decision: dict):
    """从 checkpoint 恢复 graph 执行。"""
    from deepresearch.graph import build_graph
    from deepresearch.checkpoint.manager import CheckpointManager

    graph = build_graph()
    app_graph = graph.compile(checkpointer=...)  # 需要同一 checkpointer

    try:
        async for chunk in app_graph.astream(
            Command(resume=decision),  # 👈 关键：通过 Command 恢复
            config,
        ):
            for node_name, node_result in chunk.items():
                sse_manager.put_event(task_id, "node_done", {
                    "node": node_name, ...
                })

        # 正常完成
        final_state = app_graph.get_state(config)
        # ... 保存结果 ...

    except GraphInterrupt as gi:
        # 可能再次触发审核（amel 路径会回到 research_agent → merge → human_review）
        # ...
    except Exception as e:
        # ...
```

**关键点：** `Command(resume=decision)` 将人工决策注入 `interrupt()` 的返回值，graph 从暂停点继续执行。`human_review_node` 中 `human_decision = interrupt({...})` 将收到 `decision`。

## 4. 技术注意事项

### 4.1 Checkpointer 一致性

恢复执行时必须使用**同一个 checkpointer 实例**（同一个 SqliteSaver 连接）。建议在 Server 启动时创建全局 checkpointer，或通过 session_dir 重建。

### 4.2 线程安全

HITL 恢复路径中有并发问题：
- 用户可能重复提交 review（幂等性）
- `resume_graph` 协程与 `create_task` 协程共享 task_manager

建议：
- `/review` endpoint 检查 `status == REVIEW_REQUIRED`（已实现）
- `task_manager` 使用 `threading.Lock` 保护写操作

### 4.3 前端适配

`ReviewPanel.vue` 需要在 SSE `review_required` 事件到达时自动展示。当前实现可能依赖轮询 `GET /api/tasks/{id}` 来检测 `REVIEW_REQUIRED` 状态——确认 SSE 事件是否已正确对接。

### 4.4 Amend 循环

`amend` 路径构造 `critique_result.pass=False` → `route_after_critique` 路由回 `research_agent` → 再次 merge → 再次 human_review。这意味着**同一任务可能多次触发审核**。Server 需要处理连续的 interrupt/resume 循环。
