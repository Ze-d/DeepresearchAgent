# V2.1 多 Agent 并发研究 — 设计文档

> **状态:** 设计完成 | **日期:** 2026-06-15 | **基于:** v2 Web 应用完整实现

## 1. 目标

在 v2 FastAPI + Vue 3 + SSE 架构之上，引入多 Agent 并发研究能力，按三个 Phase 推进：

| Phase | 主题 | 核心价值 | 优先级 |
|-------|------|---------|--------|
| Phase 1 | 多 Agent 并发研究 | 速度 — 4 Agent 并行扇出，缩短研究耗时 | P0 |
| Phase 2 | 交叉验证 + 结果合并 | 质量 — 多来源互相确认，发现矛盾 | P1 |
| Phase 3 | Human-in-the-Loop 审核 | 可控性 — 人工审批/补充/重做 | P2 |

**核心原则：** 每个 Phase 独立可交付，Phase 2/3 不改图拓扑（仅在节点内部升级）。

### 1.1 范围

| 在范围内 | 不在范围内 |
|---------|-----------|
| 4 个专业化 Research Agent | Agent 数量超过 4 个 |
| LangGraph Send API 并行扇出 | 分布式 Agent（多进程/多机） |
| AgentProfile 策略注入 | 每个 Agent 独立 LLM 模型 |
| Merge 节点（去重 + 交叉验证 + 排序） | Agent 间实时通信 |
| LangGraph interrupt() 人工审核 | 多轮对话式审核 |
| 可选 HITL — 默认全自动 | 审核历史/审批链 |
| 前端 ReviewPanel 组件 | 审核统计分析 |

## 3. Phase 1：多 Agent 并发研究

### 3.1 Graph 拓扑

```text
START --> plan (增强：输出 source_types)

plan --[Send API 扇出]--> research_agent × N (并发)
    ├── profile: "paper"  ─── site:arxiv.org + 学术关键词
    ├── profile: "github" ─── site:github.com + 代码关键词
    ├── profile: "blog"   ─── 技术博客偏向
    └── profile: "docs"   ─── site:docs.* + 官方文档关键词

research_agent × N --[barrier]--> merge (新增)

merge --> summary (复用 v2)
summary --> critique (复用 v2)
critique --[pass/fail]--> final | loop back
final --> END
```

### 3.3 Send API 扇出

```python
# deepresearch/graph.py

from langgraph.types import Send

def fanout_to_agents(state: AgentState) -> list[Send]:
    """根据每个 sub_question 的 source_types 创建并行 Send"""
    sends = []
    for sq in state["research_plan"]["sub_questions"]:
        for st in sq.get("source_types", ["blog"]):
            sends.append(Send("research_agent", {
                "agent_profile": st,
                "sub_question": sq,
                "user_query": state["user_query"],
            }))
    return sends

def build_graph(llm=None):
    graph = StateGraph(AgentState)
    graph.add_node("plan", make_plan_node(llm))
    graph.add_node("research_agent", make_research_agent(llm))
    graph.add_node("merge", make_merge_node(llm))
    # ... 其余节点复用
    graph.add_edge(START, "plan")
    graph.add_conditional_edges("plan", fanout_to_agents, path_map=["research_agent"])
    graph.add_edge("research_agent", "merge")
    # ...
    return graph
```

def cross_validate_evidences(evidences: list[dict], llm) -> list[dict]:

    1. LLM 语义聚类 — 识别讨论同一事实的 evidence

       - 多 Agent 一致 → confidence +0.1~0.2, cross_validated=True

## 5. Phase 3：Human-in-the-Loop 审核

### 5.1 审核节点位置

```text
Plan --> research_agent × 4 --> Merge --> [🛑 Human Review] --> Critique --> Final
                                              │
                              ┌───────────────┼───────────────┐
                              ▼               ▼               ▼
                          ✅ Approve      📝 Amend        🔄 Redo
                          继续Critique   补充搜索后       修改方向
                                         重新Research     重新开始
```

### 5.2 实现机制

```python
# nodes/human_review.py (新增)

from langgraph.types import interrupt

def make_human_review_node(llm):
    def human_review_node(state: AgentState) -> dict:
        human_decision = interrupt({
            "action": "review_requested",
            "merge_summary": state.get("merge_summary", {}),
            "sources": state.get("sources", []),
            "evidences": state.get("evidences", []),
            "conflicts": state.get("merge_summary", {}).get("conflicts", []),
        })
        return {
            "human_review": human_decision,
            "status": f"reviewed_{human_decision['action']}",
        }
    return human_review_node
```

### 5.3 三种审核动作

| 动作 | 含义 | 流程走向 |
|------|------|---------|
| `approve` | 审核通过 | 继续 Critique → Final |
| `amend` | 补充搜索 | 携带 `new_queries` 回到 Research |
| `redo` | 重做 | 携带 `revised_plan` 重新执行 |

### 5.4 API 暴露

```python
# POST /api/tasks/{task_id}/review
# Request: {"action": "approve"|"amend"|"redo", "notes": "...", "new_queries": [...]}
# Response: {"status": "resumed"}
```

### 5.5 可选性设计

- `deepresearch/config.py`: `human_review_enabled: bool = False`
- API 参数: `POST /api/tasks {query, human_review: true}` 任务级别覆盖
- 默认关闭 — 全自动路径跳过审核节点
